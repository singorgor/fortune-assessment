2026丙午·命盘运势推演录

1. 产品概述
1.1 产品名称

2026丙午·命盘运势推演录

1.2 核心价值

输入出生信息推导命盘画像（四柱、日主、五行、十神等简化结果），结合用户选择的“用事与处境”，生成 2026 年趋势与行动建议：总运 + 五领域 + 12个月节气月提示。

报告内容“爽感好读”，且每块结论都有 1–2 句“依据简述”（回指日主/五行/十神/丙午）。

1.3 技术约束

纯前端（无后端、无数据库、无登录）

Next.js（App Router）

shadcn/ui

同一设备默认只能测评一次（可回看）

2. 理论依据与产品口径（必须展示）
2.1 理论依据

采用“四柱八字（子平法）”基本框架：出生年/月/日/时 → 四柱干支 → 日主、五行、十神结构 → 推演流年趋势。

参考典籍（用于“依据页/折叠区”的书目展示）：

《渊海子平》《三命通会》《子平真诠》《滴天髓》（列为传统体系来源即可）

2.2 排盘口径（必须在页面解释）

月柱按节气分月（非农历初一）

年界采用常见口径（如以立春为界），并写明该口径

2026 为丙午流年（用于年度主题与气势解释）

输出为趋势型与建议型，不做精确断事

3. 目标用户与场景

用户：对年度运势/传统文化感兴趣、需要方向感与行动建议的人

场景：年初规划、事业/财务/感情/健康/人际某一主题的“年度导航”

9. 测评页 /test 详细需求
9.1 Step1：出生信息（必填为主）

必填：

公历出生日期（DatePicker）

出生时间：选择“具体时间”或“未知时辰”

时区：默认当前时区，可切换

校验：

日期/时间合法；若未知时辰则采用"时柱不确定"的降级逻辑：

1. 日主、年柱、月柱正常计算

2. 时柱标记为"未知"，不影响整体命盘格局判断

3. 在结果中明确提示："时辰未知，部分细节可能存在偏差"

4. 月度运势的精准度适度降低，但仍保持趋势指导性

隐私提示（固定）：

“数据仅在本地计算，不上传。”

11. 文案与安全边界（必须）

禁止：

绝对化恐吓与灾祸断语

医疗/投资/法律具体指令

政治、宗教等敏感内容判断

使用"一定会""必定""绝对"等绝对化表述

具体时辰预测（如"某月某日要小心"）

允许：

趋势表达 + 风险管理 + 可执行建议

积极正向的引导性语言

留有余地的预测（如"倾向于""可能性较大"）

术语最小化：

日主/五行/十神/丙午，每个最多 1–2 句解释，放折叠区

14. 数据存储机制优化

14.1 存储升级

使用 IndexedDB 替代 localStorage：
- 增加数据结构复杂性
- 提升数据篡改难度
- 支持更丰富的元数据

14.2 数据完整性

存储数据包含：
- attempt_token（唯一标识）
- result_snapshot（完整结果）
- profile_hash（信息hash）
- timestamp（生成时间）
- version（版本号）

14.3 访问控制

访问 /test 时检查：
- 是否存在有效结果
- 结果是否过期（可选）
- 数据完整性验证
